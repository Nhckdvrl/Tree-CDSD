# v7 树上多智能体辩论算法中文设计草稿

## 主线定位

本轮不再把“三棵投影树”作为主算法。v6.2 可以保留为强参考版本，但论文主线应回到多 agent debate：

```text
多 agent 提出局部树节点
→ 在树上定位局部冲突或假共识
→ 围绕冲突组件辩论
→ 将裁决写回树结构
→ 对终端路径做最终辩论
```

它和 Tree-of-Thoughts 的区别是：ToT 的核心是生成多个 thought 并搜索/打分；这里的核心是 agent 间的局部立场冲突、辩论裁决、以及裁决对树结构的反写。

## 当前 v7 组件

### 1. Root Contract Debate（可选消融模块）

每个 agent 先提出一个简短 reasoning contract，包括：

- 答案槽位；
- 需要验证的桥接关系；
- 哪些情况应被拒绝，例如关系漂移、停在中间实体、过早宣称缺失信息。

然后由 referee 合并为一个根部 contract。contract 不允许直接提交答案，未知终点必须保持为 `[answer]`、`[spouse]` 等占位符。

当前结论：contract 对 2Wiki/MuSiQue 有帮助迹象，但会伤害 HotpotQA，因此暂不作为默认主算法，而是保留为可消融模块 `--ldt_use_contract`。

### 2. Layerwise Proposal

在每一层，每个 agent 基于当前树前缀提出一个 atomic next-hop。候选节点先被合并为等价类，再由 relation judge 判断：

- equivalent；
- compatible；
- conflict；
- uncertain。

### 3. Adversarial Challenger

如果所有 agent 过早形成单一路径共识，系统会引入一个 challenger，要求其提出最强局部反分支。这个模块用于处理“假共识”：

- 如果 challenger 找不到有效反分支，返回空节点；
- 如果提出的是竞争候选，进入 relation/debate；
- 如果只是兼容补充事实，不直接作为互斥分支扩展。

### 4. Local Debate

局部 conflict 或 uncertain group 会触发多 agent debate。每个 agent 只围绕当前父节点下的候选 next-hop 发言，referee 决定：

- 保留哪个候选；
- 拒绝哪个候选；
- 是否多个候选仍可保留。

裁决写入候选节点的 `history.debate`，并影响后续树扩展。

### 5. Compatible Synthesis（可选消融模块）

不是所有 compatible 候选都应该成为兄弟分支。若候选是同一桥接链上的互补证据，可合成为一个 synthesis node；若只是可共存但不推进同一关系链，则不合成。

这个模块的经验结论：

- 对 HotpotQA/2Wiki 这类短桥链，synthesis 能帮助把互补事实放进同一路径；
- 对 MuSiQue，过宽 synthesis 会把无关兼容事实混进路径，导致错误；
- 因此 v7 引入 synthesis judge，只允许“链式互补”的候选合成。

当前结论：synthesis 对 HotpotQA/2Wiki 有帮助，但对 MuSiQue 和 StrategyQA 有副作用，因此暂不作为默认主算法，而是保留为可消融模块 `--ldt_use_synthesis`。

### 6. Terminal Path Debate

终端阶段不再由单一 selector 直接选答案，而是让 agents 对 top terminal paths 做 path-level debate。即使只有一条终端路径，也进行 cross-examination，检查：

- 终端答案是否回答了 contract 的答案槽位；
- 路径是否停在中间实体；
- 是否存在未验证的桥接关系；
- 是否把兼容事实误当成目标答案。

### 7. Answer-slot Grounding Debate（可选消融模块）

这一模块让 agents 在终端阶段专门围绕“最终答案是否满足问题槽位”进行辩论。它可以修复一部分替代实体/属性错误，例如 MuSiQue 中 instrument 选择从 `drums` 修到 `conga drums`。

但 full50 小矩阵显示它会明显伤害 HotpotQA，尤其会把原本正确或可接受的短答案过度审计为空或改错。因此当前默认关闭，保留为可消融模块 `--ldt_use_slot_grounding`。

## 当前实验观察

### v6.2 参考结果

v6.2 的主 full500 结果来自 Qwen3.5-9B，不是多模型平均。补齐 q3-8B 和 q25-14B 后，主模型矩阵保存在：

`analysis/ldt/v5/v62_main_model_full500_matrix.csv`

v6.2 在三个主模型上宏平均均略高于对应 v4.10 / CDSD，但它更像三投影参考系统，不应作为最终论文主算法。

### v7 q3-8B 小规模结果

当前已跑 q3-8B full50：

| 版本 | HotpotQA | 2Wiki | MuSiQue | StrategyQA | 观察 |
| --- | ---: | ---: | ---: | ---: | --- |
| default / absorb | 78 | 74 | 58 | 90 | 初版默认主候选，稳定但仍弱于 v6.2 |
| default + absence challenge | 78 | 76 | 58 | 90 | 当前默认主候选，加入缺失声明反证辩论 |
| + synthesis | 80 | 78 | 52 | 86 | 短链变强，长链被过宽合成伤害 |
| + contract + synthesis | 74 | 80 | 60 | 88 | 复杂桥链变强，HotpotQA 受损 |
| contract as judge only | 78 | 78 | 52 | 86 | 没有保住 MuSiQue 收益 |

对应 32B judge 小样本：

| 版本 | HotpotQA | 2Wiki | MuSiQue | StrategyQA |
| --- | ---: | ---: | ---: | ---: |
| q3-8B default / absorb | 80 | 76 | 56 | 90 |
| q3-8B default + absence challenge | 80 | 78 | 56 | 90 |
| q25-14B default + absence challenge | 74 | 74 | 54 | 90 |
| q3-8B + contract + synthesis | 78 | 82 | 54 | 88 |
| q3-8B + numeric final fix | 80 | 78 | 56 | 90 |
| q25-14B + numeric final fix | 76 | 74 | 52 | 90 |

结论：v7 当前主候选的结构更符合多 agent debate 题目，但性能还没有达到最终要求，尤其 q25-14B 上泛化不足。后续必须继续从失败轨迹中提炼算法层修复。

slot grounding 的 local full50 结果：

| 模型 | HotpotQA | 2Wiki | MuSiQue | StrategyQA | 结论 |
| --- | ---: | ---: | ---: | ---: | --- |
| q3-8B + slot grounding | 68 | 78 | 58 | 90 | 修个别 MuSiQue，但 HotpotQA 明显下降 |
| q25-14B + slot grounding | 64 | 72 | 58 | 92 | StrategyQA 上升，但 HotpotQA 下降严重 |

numeric final fix 说明：早期版本复用了树节点解析里的 `_clean_answer`，它会把 QA 中的纯数字答案清空，导致年份答案如 `1999`、`1838`、`1962` 在 trace 中正确但最终输出为空。v7 当前修复为终端答案专用 cleaner，允许 QA 的纯数字答案。这是算法实现规范修复，不是 benchmark case rule。

## 当前失败模式

1. 假共识：所有 agent 走到同一错误路径，普通终端 debate 只会确认错误。
2. 兼容但不链式相关：多个事实可以同时为真，但不应合成到同一推理路径。
3. 替代实体污染：例如同一团体中多个成员都能对应不同 instrument，必须通过 contract 判断目标实体。
4. 过早缺失声明：模型说“not mentioned”时，如果桥接链还没穷尽，应进入 adversarial challenge，而不是直接终止。

## 下一步

1. 默认保留 default / absorb 作为 v7 主候选。
2. 继续从 MuSiQue 和 StrategyQA 失败轨迹中找算法层修复，重点是“替代实体”和“过早缺失声明”。
3. 对 contract 和 synthesis 做消融，不把它们混入默认主算法。
4. 下一轮继续攻“证据槽位 grounding”，但不能采用当前这种无条件终端 slot grounding；需要更局部、更少副作用的触发条件。
